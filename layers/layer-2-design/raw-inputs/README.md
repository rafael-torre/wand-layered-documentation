# Raw Inputs — Layer 2: Design

Materials gathered, not authored. Work through this checklist at the start of design work to capture the context needed for synthesis. Not every item will be available for every project — gather what exists and note what's missing.

---

## Upstream Layer Documents

The final documents from Layer 1 (Product) are the primary inputs to this layer. These are not optional — they provide the product context that shapes every design decision.


| Input | Description | Source |
| ----- | ----------- | ------ |
| Product Brief | Problem space, target users, product vision, scope | Layer 1 final |
| Feature Specs | Feature behavior, business rules, acceptance criteria, edge cases | Layer 1 final |
| User Personas | Who the users are, their goals, pain points, behaviors, technical proficiency | Layer 1 final |
| User Journeys | Cross-feature flows showing how users accomplish goals through the product | Layer 1 final |
| Success Metrics | Product KPIs and user success criteria that design decisions should serve | Layer 1 final |
| Domain Glossary | Shared vocabulary to use consistently across design assets and documentation | Layer 1 final |

## Figma Files

Figma is the visual source of truth for this layer. Files live outside the repo — capture stable URLs here so they are traceable alongside the documentation.


| Input | Description | Figma URL |
| ----- | ----------- | --------- |
| Wireframes | Low-fidelity explorations of layout and structure | |
| High-fidelity mockups | Final visual designs, screen by screen | |
| Prototypes | Interactive flows showing transitions and interactions | |
| Design system / component library | Shared components, variables, and design patterns for this project | |


> **Note on Figma URLs:** Use deep links to specific pages or frames where possible, not just the file root. This keeps references meaningful as files grow. Mark each link with the date it was last verified as current.

## User Research

Inputs from research activities that inform design decisions. These are raw — synthesis happens in the intermediate artifacts.


| Input | Description | Who Provides |
| ----- | ----------- | ------------ |
| User research session recordings | Interviews or contextual inquiries with target users | Designer / Research |
| Usability test recordings and notes | Observed sessions testing specific designs or prototypes | Designer / QA |
| Survey results | Quantitative feedback on user preferences or pain points | Designer / PM |
| Analytics and behavioral data | Usage patterns, drop-off points, heatmaps from existing products | PM / Analytics |


---

## Client-Provided

Assets and constraints provided directly by the client that shape the visual direction.


| Input | Description | Who Provides |
| ----- | ----------- | ------------ |
| Brand guidelines | Color palette, typography, logo usage, tone of voice | Client |
| Style guides | Visual standards for existing products or marketing materials | Client |
| Existing UI assets | Screenshots, design files, or style sheets from current products | Client / Designer |
| Accessibility requirements | Client-specified compliance level (WCAG AA/AAA) or platform requirements | Client / PM |


---

## Reference Materials

External inputs that inform design direction without being project-specific.


| Input | Description | Who Provides |
| ----- | ----------- | ------------ |
| Competitor UX analysis | Teardowns of how competing products handle similar features | Designer / PM |
| Design pattern references | Industry patterns for interaction models relevant to this product | Designer |
| Inspiration and benchmarks | Visual references, design systems from other products used as reference | Designer |
